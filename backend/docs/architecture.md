# Backend Architecture Notes

## Layering

PartnerLedger OS backend follows this path:

`Controllers -> Services -> Core Engines -> Repositories -> Prisma -> PostgreSQL`

The goal is to keep transport, orchestration, business logic, and persistence responsibilities separate so modules can grow without becoming tightly coupled.

## Event-Driven Flow

When a transactional action happens, the application will follow this pipeline:

1. A feature service persists the new domain record.
2. The service emits a domain event through `EventBusService`.
3. `WalletEngine` reacts to `sale.created` or `expense.created` and updates balances atomically.
4. `WalletEngine` emits `wallet.updated` after balance changes succeed.
5. `SettlementEngine` reacts to `wallet.updated`, recalculates live debts, and persists settlement suggestions.
6. `InsightEngine` and `AlertEngine` evaluate analytical rules and persist derived records.
7. WebSocket bridge services fan those changes out to connected dashboards.
8. Reporting and logs modules can subscribe without changing the original write path.

## Realtime Strategy

Socket.IO is exposed via Nest's gateway layer. The gateway is intentionally thin:

- Gateway handles connected clients and outbound broadcasts.
- Event subscribers translate domain events into socket payloads.
- Feature modules do not directly depend on websocket transport for their core behavior.

Current socket broadcasts:

- `transaction.created`
- `dashboard.update`
- `settlement.updated`
- `insight.generated`
- `alert.triggered`

## Database Modeling

The initial Prisma schema includes the required SaaS domain entities:

- `User`
- `RefreshToken`
- `Partner`
- `Wallet`
- `Sale`
- `Expense`
- `Settlement`
- `DailyReport`
- `Insight`
- `Alert`
- `Log`

Money fields use `Decimal` columns to avoid floating-point errors.

## Current Engine Status

- `WalletEngine` is live and updates `cashBalance`, `onlineBalance`, and `totalBalance`.
- `SettlementEngine` is live and recomputes a minimized transfer plan in a persistent `live-current` batch.
- `InsightEngine` stores lightweight daily insights.
- `AlertEngine` stores daily and settlement-driven alerts.

## Read Layer Status

- `DashboardService` exposes aggregated financial totals, partner balances, and recent transactions.
- `SettlementService` exposes the latest live settlement suggestions generated by the engine layer.
